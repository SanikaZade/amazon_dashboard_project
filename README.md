
# Amazon Sales Dashboard

A fully interactive, browser-based Business Intelligence dashboard built on 128,017 Amazon India orders (March – June 2022). 
---

## What This Project Does

This dashboard lets you explore Amazon India sales data visually and interactively — without writing a single line of code. You can:

- Filter orders by date, category, state, city, size, fulfilment type, order status, and more
- See KPIs update live as you apply filters
- Drill into geography, products, fulfilment, customer segments, and AI-generated insights
- Export chart data as CSV files
- Switch between a clean white theme and a dark theme

Everything runs entirely in your browser. There is no server, no database, and no internet connection required after the page loads.

---

## Tech Stack

| Layer | Tool | Why It Was Used |
|---|---|---|
| **Data processing** | Python 3 + pandas + NumPy | Reads the raw CSV, cleans it, and computes all aggregates |
| **Charts** | Plotly.js (vendored, offline) | 28 interactive charts — hover, zoom, pan, download as PNG |
| **Frontend** | Plain HTML + CSS + Vanilla JavaScript | No framework needed; keeps the project simple and portable |
| **Fonts** | Inter (Google Fonts) | Clean, modern sans-serif used across the entire UI |
| **Data format** | JSON (embedded inside HTML) | Lets the dashboard work as a single self-contained file |
| **Filter engine** | Custom vectorised JS | Runs cross-filters across 128k rows in under 50ms in the browser |

### No external dependencies at runtime

Plotly.js is saved locally in the `vendor/` folder. Once `dashboard.html` is built, it works completely offline. The only thing that requires internet is the Inter font — if you are offline, the browser will fall back to the system sans-serif font, and everything else still works fine.

## Project Structure

```
amazon_dashboard_project/
│
├── dashboard.html              ← The finished dashboard. Open this in your browser.
│
├── dashboard_template.html     ← The HTML/CSS/JS source. Data is injected into this.
│
├── build_dashboard.py          ← Main build script. Reads CSV → writes JSON → writes HTML.
│
├── inject_data.py              ← Quick helper: just re-injects existing JSON into template.
│
├── requirements.txt            ← Python dependencies (pandas, numpy)
│
├── data/
│   ├── amazon_clean_Data.csv   ← Source dataset (128,017 rows, ~29 MB)
│   └── dashboard_data.json     ← Pre-aggregated data (~6 MB), generated by build script
│
└── vendor/
    └── plotly.min.js           ← Plotly charting library saved locally (offline use)
```

### What each file does

**`dashboard.html`** — This is the file you actually open. It is a single HTML file with all charts, all data, and all JavaScript inside it. You can share this one file and the recipient can open it without installing anything.

**`dashboard_template.html`** — The source code of the dashboard before data is injected. It contains the HTML layout, all CSS styling, the Plotly chart-drawing code, and a placeholder `__DATA_JSON__` where the data gets inserted during the build step.

**`build_dashboard.py`** — The main Python script. Run this whenever you update the CSV. It reads the CSV, computes all the numbers (totals, averages, breakdowns by category/state/date etc.), saves those numbers to `dashboard_data.json`, and then injects them into the template to produce `dashboard.html`.

**`inject_data.py`** — A faster shortcut for when you have already run the full build and only want to re-apply the data to a changed template, without re-reading the CSV. Useful during template development.

## How It Works — The Full Workflow

Understanding the pipeline from raw data to live dashboard:

```
amazon_clean_Data.csv
        │
        ▼
  build_dashboard.py          ← You run this with Python
  (pandas + numpy)
        │
        │  Reads all 128,017 rows
        │  Computes ~25 aggregations:
        │    - daily & monthly revenue trends
        │    - orders by category, size, state, city
        │    - cancellation rates by fulfiment/state
        │    - order value distribution
        │    - B2B vs B2C split
        │    - week × day heatmap
        │    - and more...
        │
        ▼
  dashboard_data.json         ← Saved as a structured JSON file
        │
        ▼
  dashboard_template.html     ← The HTML/JS template
  (placeholder replaced)
        │
        ▼
  dashboard.html              ← Final output — open in browser
```

### What happens inside the browser

When you open `dashboard.html` in your browser:

1. The page loads and JavaScript reads the embedded JSON data (the full dataset and all aggregates).
2. The filter engine indexes all 128,017 transaction rows in memory using compact integer arrays — this is why filtering feels instant even with so many rows.
3. When you interact with any filter (date range, category, state, etc.), the engine re-scans all rows in a tight loop, collects the matching indices, recomputes the affected aggregates, and redraws only the charts on the visible tab. This all happens in under 50 milliseconds.
4. Charts are drawn by Plotly.js — each one is declared in a few lines of JavaScript that pass the aggregated data and a layout configuration.

## How to Run It

### Option A — Just open the dashboard (no setup needed)

If you received the project with `dashboard.html` already built:

```
Double-click dashboard.html → opens in your default browser
```

That's it. No installation, no server, no Python required.

### Option B — Rebuild after updating the data

Use this if you replace the CSV with new data, or if you want to make changes to the build script.

**Step 1 — Install Python dependencies**

```bash
pip install -r requirements.txt
```

This installs pandas and numpy. You only need to do this once.

**Step 2 —  the dataset**

If you have a new CSV file, replace `data/amazon_clean_Data.csv` with your file. Make sure the column names match the original schema (see Dataset Details below).

**Step 3 — Run the build script**

```bash
python build_dashboard.py
```

This will:
- Read and process the CSV (takes a few seconds for 128k rows)
- Write the updated `data/dashboard_data.json`
- Generate the updated `dashboard.html`

**Step 4 — Open the result**

```
Open dashboard.html in your browser
```

### Option C — Quick rebuild 

If you only edited `dashboard_template.html` and want to apply existing data without re-reading the CSV:

```bash
python inject_data.py
```

This is faster because it skips the pandas processing step.

## Dashboard Sections

The dashboard has six tabs at the bottom of the screen:

### Overview
The landing view. Shows:
- Daily revenue and order count trend line (dual-axis)
- Monthly revenue bar chart
- B2B vs B2C split (donut chart)
- Orders by day of the week
- Week × day orders heatmap (which week of the year, which day had most orders)

### Products & Sales
Everything about what was sold:
- Revenue by category (horizontal bar chart, click to cross-filter)
- Units sold by category (volume contribution)
- Size distribution (how many XS / S / M / L / XL / XXL etc. were ordered)
- Category × size heatmap (which sizes sell in which categories)
- Top 15 category-size combinations (most popular pairings)
- Return rate by category

### Geography
Where the orders went:
- Interactive India map — states shaded by revenue intensity, click to cross-filter
- Revenue by state — Top 10 states (horizontal bar chart, click to cross-filter)
- Cancellation rate by state (highlights problem regions in red)
- B2B share by state (top 12 states)

### Fulfillment
How orders were processed:
- Amazon-fulfilled vs Merchant-fulfilled split (donut chart)
- Cancellation rate comparison: Amazon vs Merchant (bar chart)
- Shipping level distribution (Standard, Expedited, etc.)
- Order status split by fulfillment type (stacked bar chart)
- Return rate by category

### AI Insights
Business findings and raw data explorer, all derived live from the filtered data:
- AI natural language insights — key metric observations auto-generated from current filter state
- Executive summary — written business narrative of performance
- Raw records explorer — paginated live transaction table showing every individual order matching the active filters (date, category, size, amount, state, city, fulfilment, status columns)
- All text and numbers update automatically when you apply or remove filters

### Advanced
Deep-dive charts for analysts (10 charts total):
- Revenue treemap — category to size, proportional revenue breakdown
- Order sunburst — category to order status, hierarchical distribution
- State performance bubble chart — X = orders, Y = revenue, bubble size = cancel rate
- Scatter plot — order amount vs quantity, coloured by category
- Fulfillment funnel — orders placed → shipped → delivered → cancelled
- Cancellation rate gauge — needle gauge against the 20% target threshold
- Revenue waterfall by category — cumulative contribution of each category
- Category radar chart — multi-KPI comparison (revenue, qty, AOV, cancel rate, return rate)
- Sankey flow diagram — category → fulfillment type → order status (full lifecycle)
- Calendar heatmap — daily order volume as a bar chart coloured by intensity

## Features

### Cross-filtering
Every filter applied (date range, category, state, etc.) updates all visible charts simultaneously. Filter to "T-shirts only in Maharashtra" and the KPI cards, all charts, and insight text update to reflect just those orders.

### Sidebar filters
Click the **Filters** button in the top left to open the filter panel. Available filters:
- Date range (from / to)
- Product category (multi-select)
- Indian state (multi-select)
- City (text search)
- Size (multi-select)
- Fulfilment type (Amazon / Merchant)
- Order status (Shipped / Cancelled / Returned / Pending)
- B2B toggle (Business vs Consumer orders)
- Courier status
- Sales channel

### Active filter tags
When filters are active, coloured tags appear below the header showing what is currently filtered. Click the × on any tag to remove that filter.

### Chart CSV export
Every chart card has a small **↓ CSV** button in the top-right corner. Clicking it downloads that specific chart's data as a CSV file, ready to paste into Excel or any other tool.

### Theme toggle
The 🌙 button in the header switches between white (default) and dark theme. Your preference is saved in the browser so it persists across refreshes.

### Responsive layout
The dashboard adapts to different screen sizes. Charts reflow from 4-column to 2-column to 1-column on narrower screens.

## Dataset Details

**File:** `data/amazon_clean_Data.csv`
**Rows:** 128,017 orders
**Period:** March – June 2022 (Q2 2022)
**Source:** Amazon India sales export

### Key columns used by the build script

| Column | Description |
|---|---|
| `Order_ID` | Unique order identifier |
| `Date` | Order date (YYYY-MM-DD) |
| `Category` | Product category (T-shirt, Shirt, Blazzer, etc.) |
| `Size` | Product size (XS, S, M, L, XL, XXL, 3XL, Free) |
| `Qty` | Quantity ordered |
| `Amount` | Order value in Indian Rupees |
| `Amount_Zero_Flag` | True if the order had zero value (excluded from revenue KPIs) |
| `Status_Group` | Simplified status: Shipped / Cancelled / Returned / Pending |
| `Fulfilment` | Amazon or Merchant |
| `Shipping_Level` | Expedited / Standard / etc. |
| `B2B` | True = Business order, False = Consumer order |
| `Sales_Channel` | Amazon.in or other |
| `Courier_Status` | Shipped / Unshipped / Cancelled |
| `Ship_State` | Destination state (India) |
| `Ship_City` | Destination city |
| `Order_Month` | Month name (March, April...) |
| `Order_MonthNum` | Month number for sorting |
| `Order_DayOfWeek` | Monday, Tuesday... |
| `Order_Week` | ISO week number |
| `Order_Value_Segment` | Low / Medium / High / Premium (derived bucket) |

If you replace the CSV, ensure these column names are present. The build script will raise a KeyError immediately if any required column is missing.

## Customisation & Extending

### Changing the colour theme
Open `dashboard_template.html` and edit the CSS variables at the top of the `<style>` block (inside `:root { ... }`). The key variables are `--bg`, `--panel`, `--ink`, `--muted`, and `--accent`.

After editing the template, run `python inject_data.py` to rebuild the dashboard without re-processing the CSV.

### Adding a new chart
1. Add a `<div id="chartMyNew"></div>` inside the correct section in `dashboard_template.html`
2. Write a `Plotly.newPlot('chartMyNew', [...], L({...}), CFG)` call in the relevant render function in the JavaScript section
3. (Optional) Add a new aggregate to `build_dashboard.py` if the chart needs data not already in `dashboard_data.json`, then re-run `python build_dashboard.py`

### Using a different dataset
As long as your CSV has the same column names, simply replace `data/amazon_clean_Data.csv` and run:
```bash
python build_dashboard.py
```
All charts, KPI cards, and insight text will update automatically to reflect the new data. No JavaScript changes are needed.

## Troubleshooting

**The page opens but shows no charts**
Make sure `vendor/plotly.min.js` is in the same folder as `dashboard.html`. If it is missing, the charts cannot render.

**Charts look wrong after replacing the CSV**
Run `python build_dashboard.py` (not just `inject_data.py`) to regenerate `dashboard_data.json` from the new CSV. Using `inject_data.py` only re-injects the old JSON.

**Python error: KeyError on a column name**
Your replacement CSV has a differently named column. Check the Dataset Details section above and rename the column in your CSV (or edit the column reference in `build_dashboard.py`).

**Filters seem slow**
This is unlikely with the current dataset size, but if your CSV has several million rows, the in-browser filter engine may take longer. The filter loop is already vectorised; the main bottleneck would be Plotly re-rendering 28 charts. You could limit the number of active charts per tab to improve performance.

**Theme preference not saving**
The theme is saved to `localStorage` in your browser. If you are opening the file from a network path or certain browser configurations block `localStorage`, the preference won't persist between refreshes. This is a browser restriction, not a bug in the dashboard.

**Font looks different**
The Inter font is loaded from Google Fonts, which requires an internet connection on first load. If you are offline, the browser uses your system's default sans-serif font. The dashboard looks slightly different but works completely.
