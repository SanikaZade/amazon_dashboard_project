Amazon Sales Dashboard — Live Data Project

This dashboard analyzes 128,017 real Amazon India orders from March–June 2022 to answer a practical business question: where is revenue coming from, and where is it leaking? Every number on every chart is computed live from the underlying dataset — there are no static images, mock numbers, or "coming soon" placeholders.

Problem it solves

Sellers and ops teams need a fast way to see revenue trends, which product categories and sizes perform best, where orders get cancelled or returned, and how fulfilment method (Amazon vs. Merchant) affects that — without opening a spreadsheet. This dashboard turns a 128k-row CSV into an interactive, drill-able view of all of that in one page.

Tech stack

Python (pandas / numpy) HTML / CSS / vanilla JS Plotly.js Static single-file build

Key features

  • 23 charts across 7 sections, all rendered from a single aggregated JSON payload
  • Live search filtering across Products & Geography charts (category, state, city)
  • One-click CSV export on every chart card, for pulling numbers into Excel/Sheets
  • Auto-hide safety net: any chart that ends up with no real data disappears instead of showing a blank box
  • Fully responsive layout, dark theme, tabbed navigation

How it's built

build_dashboard.py reads the CSV, computes all aggregates with pandas, writes them to data/dashboard_data.json, then injects that JSON into dashboard_template.html to produce this self-contained dashboard.html. Swap in a new CSV with the same schema and re-run the script to refresh every number.
